🧱 1. API Gateway — Spring Cloud Gateway
Tech: Spring Boot + Spring Cloud Gateway
✅ API Gateway
* Checks if a request has a valid JWT token

* Checks if token is expired

* May check permissions (RBAC) if delegated

* Rejects unauthenticated requests

* Routes valid requests to microservices

* Acts as a gatekeeper

❌ Gateway does NOT:
   * Authenticate the user with database

   * Store users

   * Manage passwords

   * Generate JWT tokens

   * Refresh tokens

   * Handle login or register


Responsibilities:
      * Entry point for all client requests

      * Route requests to internal services

      * Rate limiting / Circuit breaking

      * Token validation

      * Logging & load balancing

Why Spring?
         * Best ecosystem for gateways

         * Built-in resilience via Resilience4j

         * Easy JWT validation

________________

🧱 3. Product Service — Spring Boot
Tech: Spring Boot + MongoDB
Responsibilities:
                     * Manage products

                     * Filtering, sorting, categories

                     * Product metadata

                     * Images support

Why Spring?
                        * Easy CRUD dev

                        * Great support for MongoDB
Database: MongoDB (flexible for product data)
brew services start mongodb-community
________________


🧱 4. Cart Service — Node.js
Tech: Node.js + Express + Redis
Responsibilities:
                           * Add/remove items to cart

                           * Store items temporarily

                           * Quick reads/writes

Why Node.js?
                              * Non-blocking and very fast for I/O

                              * Cart operations need low latency

Database: Redis (in-memory → ultra-fast)
________________


🧱 5. Order Service — Go (Golang)
Tech: Go + gRPC (optional) + PostgreSQL
Responsibilities:
                                 * Create orders

                                 * Validate payment status

                                 * Handle concurrency

                                 * Orchestrate inventory & payment checks

                                 * Emit events (OrderCreated, OrderCompleted)

Why Go?
                                    * Amazing concurrency (goroutines)

                                    * High throughput for order processing

Database: PostgreSQL (orders need ACID consistency)
________________


🧱 6. Payment Service — Go
Tech: Go + PostgreSQL
Responsibilities:
                                       * Payment simulation

                                       * Wallet balance mock

                                       * Call external payment API (optional)

                                       * Emit PaymentSuccess or PaymentFailed

Why Go?
                                          * Low latency

                                          * High reliability

________________


🧱 7. Inventory Service — Spring Boot
Tech: Spring Boot + MongoDB
Responsibilities:
                                             * Stock tracking

                                             * Stock update after order

                                             * Sync inventory

Why Spring?
                                                * Easy domain modeling

                                                * Strong consistency with eventual event consumption

________________


🧱 8. Notification Service — Node.js
Tech: Node.js + Kafka consumer + Nodemailer / Twilio
Responsibilities:
                                                   * Sends email/SMS on:

                                                      * Order placed

                                                      * Payment success

                                                      * Delivery updates

Why Node.js?
                                                         * Perfect for lightweight async I/O tasks

________________

🌐 12. Communication Patterns
                                                                  * Sync communication (HTTP REST):

                                                                     * API Gateway → Services

                                                                     * Cart → Product (get product info before adding)

                                                                        * Async communication (Kafka events):

                                                                           * Order → Inventory

                                                                           * Inventory → Order

                                                                           * Order → Payment

                                                                           * Payment → Notification

This makes your system fault-tolerant.
